## 零、游戏世界观与背景设定

### 0.1 时代背景

**时间：** 丝绸之路繁盛时期（约公元100-200年）
**地点：** 从长安到罗马的商路

**世界观设定：**
- 这是一个诸国并立、丝路繁荣的时代
- 商队是最重要的文明传播者
- 每一支商队都可能改变国家的命运

### 0.2 势力分布

| 势力 | 特点 | 代表地点 |
|------|------|----------|
| 汉朝 | 强盛富饶，丝绸产地 | 长安、洛阳 |
| 安息 | 商路中转，能工巧匠 | 泰西封 |
| 贵霜 | 佛教传播，艺术繁荣 | 犍陀罗 |
| 罗马 | 超级大国，奢侈品需求 | 罗马、亚历山大 |
| 草原部落 | 骑兵天下，劫掠为生 | 塞外 |

### 0.3 主角设定

**身份：** 丝绸之路上的年轻行商
**背景：** 继承了家族的商队和一份神秘的"商道秘籍"
**目标：** 成为丝路传奇，建立不世功业

## 一、游戏愿景与核心体验

### 1.1 一句话描述

> "一张铺在桌面上的古代卷轴，一位行商的传奇之旅。键盘敲击化作战马蹄声，鼠标移动化作校准方向，在透明的悬浮窗中，你的商队正穿越千年风沙。"

### 1.2 核心体验关键词

| 关键词 | 描述 |
|--------|------|
| **沉浸感** | 悬浮窗如输入法般安静陪伴，不打断工作 |
| **成就感** | 指数级成长曲线，每次上线都有惊喜 |
| **策略性** | 资源分配、路线选择、队伍配置 |
| **故事性** | 分支叙事事件，每趟旅程都是独特故事 |
| **收集欲** | 卡牌收集、角色收集、成就收集 |

### 1.3 差异化定位

对比主流放置游戏，本作独特定位：

| 维度 | 主流放置游戏 | 桌面旅者 |
|------|-------------|----------|
| 视觉 | 全屏游戏 | 透明悬浮窗，桌面融合 |
| 交互 | 纯挂机 | 虚实结合，操作感 |
| 叙事 | 无 | 分支事件树，Roguelike |
| 目标 | 无尽刷 | 旅程终点 + Prestige |
| 社交 | 公屏聊天 | 异步PVP（延迟对战） |

## 二、核心玩法循环

### 2.1 宏观循环架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         核心循环三层                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐          │
│   │   短循环    │ ──▶ │   中循环    │ ──▶ │   长循环    │          │
│   │  (分钟级)   │     │  (小时级)   │     │  (天级)     │          │
│   └─────────────┘     └─────────────┘     └─────────────┘          │
│         │                   │                   │                  │
│         ▼                   ▼                   ▼                  │
│   键盘/鼠标输入        节点旅行事件         旅程阶段/Prestige        │
│   触发即时事件        资源获取/消耗        角色成长/重置             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 短循环：虚实交互（分钟级）

**触发条件：** 玩家键盘输入或鼠标移动

**玩法流程：**
```
键盘敲击 ──▶ 脚程累积 ──▶ 旅行进度加速 ──▶ 节点到达
                │
                ▼
           校准判定 ──▶ 成功：额外奖励
                    ──▶ 失败：资源损耗
```

**具体设计：**

| 输入类型 | 转化效率 | 额外效果 |
|----------|----------|----------|
| 键盘敲击(WASD/方向键) | 每10次 = 1秒脚程 | 连击加成（10次/秒+ = 1.5x） |
| 鼠标移动 | 每100px = 0.1秒脚程 | 精准校准（静止>2秒后移动 = 2x） |
| 滚轮滚动 | 每次 = 0.5秒脚程 | 无 |

**校准系统：**
- 旅行方向与地图方向匹配判定
- 成功：获得"校准点数"，可用于强化下一次旅行
- 失败：消耗额外脚程

### 2.3 中循环：节点旅行（小时级）

**触发条件：** 自动推进或主动交互

**玩法流程：**
```
开始旅程 ──▶ 节点移动 ──▶ 事件触发 ──▶ 选择结果 ──▶ 奖励/惩罚
     │                                              │
     └─────────── 循环直到旅程终点 ◀────────────────┘
```

**节点类型与事件：**

| 节点类型 | 出现概率 | 事件类型 | 预期收益 |
|----------|----------|----------|----------|
| 起点 | 固定 | 无 | - |
| 古城(City) | 15% | NPC遭遇/商店 | 中等（金币/道具） |
| 集市(Market) | 15% | 交易事件 | 低（但稳定） |
| 荒野(Wilderness) | 20% | 战斗/随机 | 高波动 |
| 副本(Dungeon) | 25% | 连续战斗 | 高（但耗时） |
| BOSS | 10% | 极限战斗 | 极高（一次性） |
| 事件点(Event) | 10% | 分支叙事 | 极高（看选择） |
| 终点 | 固定 | 阶段结算 | - |

### 2.4 长循环：旅程与重置（天级）

**旅程阶段：**
```
第一章：春风得意 ──▶ 第二章：江湖险恶 ──▶ 第三章：风云际会 ──▶ ... ──▶ 最终章：落叶归根
   10节点              15节点               20节点                 循环
```

**Prestige系统（轮回）：**
- 每次到达终点后，可选择"轮回"
- 轮回消耗所有资源，保留：
  - 角色等级（部分）
  - 已解锁的被动技能
  - 成就进度
  - 卡牌收藏
- 轮回奖励：轮回点数，可兑换强力永久buff

## 三、成长系统

### 3.1 角色成长体系

**六维属性：**

| 属性 | 简称 | 效果 | 成长方式 |
|------|------|------|----------|
| 体力(Vitality) | VIT | 最大HP，决定续航 | 食物/休息 |
| 攻击(Attack) | ATK | 伤害输出 | 武器/战斗 |
| 防御(Defense) | DEF | 减伤承受 | 护甲/休息 |
| 速度(Speed) | SPD | 行动顺序/旅行速度 | 装备/事件 |
| 魅力(Charisma) | CHA | NPC态度/交易折扣 | 对话/事件 |
| 智慧(Wisdom) | WIS | 事件判断/暴击率 | 书籍/事件 |

**升级机制：**
- 每次战斗胜利获得经验值
- 经验值满了升级，属性全面提升
- 等级上限：100级（Prestige后可突破）

### 3.2 装备系统

**装备槽位：**

| 槽位 | 装备类型 | 稀有度影响 |
|------|----------|------------|
| 武器 | 单手/双手/远程 | 伤害值 |
| 护甲 | 轻甲/中甲/重甲 | 防御值+速度惩罚 |
| 饰品1 | 戒指/项链/护符 | 特殊属性 |
| 饰品2 | 戒指/项链/护符 | 特殊属性 |
| 坐骑 | 马/骆驼/马车 | 旅行速度+容量 |

**装备品质：**
- 普通(white) → 优秀(green) → 稀有(blue) → 史诗(purple) → 传说(orange)
- 品质影响基础属性值和附加词缀数量

### 3.3 技能系统

**技能分类：**

| 类型 | 获取方式 | 数量上限 |
|------|----------|----------|
| 被动技能 | 事件解锁/升级 | 4个 |
| 主动技能 | 事件解锁/购买 | 2个 |
| 终极技能 | Prestige解锁 | 1个 |

**示例技能：**

| 技能名 | 类型 | 效果 |
|--------|------|------|
| 商旅之道 | 被动 | 交易事件收益+30% |
| 疾风步 | 被动 | 旅行速度+20% |
| 横扫千军 | 主动 | 对所有敌人造成50%伤害 |
| 财源广进 | 主动 | 消耗生命值换取金币（1HP=10金币） |
| 轮回之力 | 终极 | Prestige时保留50%资源（原为0） |

### 3.4 队伍系统

**队伍配置：**
- 最多6人同时旅行（代码默认maxTeamSize=6）
- 每人负责不同职责

**职业系统：**

| 职业 | 职责 | 专属属性 | 专属技能 |
|------|------|----------|----------|
| 商贾 | 交易 | 交易价格+20% | 议价 |
| 镖师 | 战斗 | 伤害+15% | 护卫 |
| 学者 | 探索 | 事件触发率+30% | 鉴定 |
| 医者 | 支援 | 队伍HP回复+50% | 治疗 |

> **注意**：当前TeamMemberData暂无职业字段，职业系统为规划功能，待代码实现

**NPC招募：**
- 通过事件遇到NPC
- 消耗金币招募
- 招募后解锁该NPC专属职业

## 四、资源与经济系统

### 4.1 资源类型

| 资源 | 用途 | 获取方式 | 消耗场景 |
|------|------|----------|----------|
| 金币(Gold) | 通用货币 | 战斗/交易/事件 | 交易/招募/强化 |
| 脚程(Steps) | 旅行燃料 | 时间积累/键盘输入 | 节点移动 |
| 食物(Food) | HP恢复 | 购买/事件 | 旅行消耗 |
| 疲劳度(Energy) | 行动限制 | 时间恢复/休息 | 高难度事件 |
| 轮回点数(Prestige) | 永久加成 | 轮回重置 | 技能/buff |

### 4.2 经济循环

```
                    事件奖励
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐
   │ 战斗收益 │    │ 交易收益 │    │ 事件奖励 │
   └────┬────┘    └────┬────┘    └────┬────┘
        │               │               │
        └───────────────┼───────────────┘
                        ▼
                 ┌─────────────┐
                 │    背包     │
                 └──────┬──────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐
   │  装备   │    │  消耗   │    │  出售  │
   │ 强化   │    │ 旅行   │    │  回血  │
   └─────────┘    └─────────┘    └─────────┘
```

### 4.3 交易系统

**NPC商店：**
- 古城和集市有商店
- 刷新时间：每6小时
- 特殊商品：限时折扣

**玩家间交易：**
- 延迟PVP系统的衍生
- 玩家可上架物品
- 另一玩家购买（异步）

### 4.4 离线收益

**计算公式：**
```
离线收益 = 在线收益 × 时间系数 × 离线加成

时间系数：
- <1小时：1.0x
- 1-6小时：0.8x
- 6-24小时：0.5x
- >24小时：0.2x

离线加成：来自被动技能和Prestige buff
```

**积压事件：**
- 离线期间每60秒可能触发1个事件
- 上线时统一展示事件结算

## 五、事件系统

### 5.1 事件分类

**即时事件（EventQueue）：**
- 触发：节点到达、旅行中随机
- 特点：快速结算，选择少

**分支叙事（EventTree）：**
- 触发：特定节点/条件满足
- 特点：多选择、多结局

### 5.2 事件类型详解

| 事件类型 | 触发频率 | 典型选择 | 结果影响 |
|----------|----------|----------|----------|
| 战斗(Combat) | 高 | 正面硬刚/逃跑/求和 | 金币/经验/损失 |
| 交易(Trade) | 中 | 买/不买/砍价 | 道具/金币 |
| 发现(Discovery) | 中 | 探索/忽略/谨慎探索 | 宝箱/陷阱 |
| 遭遇(Encounter) | 中 | 交谈/战斗/贿赂 | NPC好感/任务 |
| 故事(Story) | 低 | 无（纯叙事） | 剧情解锁 |
| 神秘(Mystery) | 低 | 赌博选择 | 高风险高回报 |

### 5.3 分支叙事设计

**五棵事件树示例：**

| 事件树 | 节点数 | 选择数 | 可能结局 |
|--------|--------|--------|----------|
| 新手教程 | 5 | 8 | 3 |
| 森林探险 | 10 | 15 | 5 |
| 宝藏猎手 | 15 | 22 | 7 |
| 英雄传承 | 20 | 30 | 10 |
| 黑暗秘密 | 25 | 40 | 12 |

**叙事风格：**
- 第一章：轻松愉快，新手引导
- 第二章：江湖险恶，抉择艰难
- 第三章：风云际会，多线叙事
- 终章：落叶归根，情感收尾

### 5.4 事件触发规则

```
事件触发判定：
1. 检查节点是否有预设事件 ──▶ 有：触发预设
2. 检查事件树条件 ──▶ 满足：触发事件树
3. 随机事件判定 ──▶ 骰子点数 < 事件概率
4. 触发即时事件
```

## 六、延迟PVP系统

### 6.1 核心机制

**类似皇室战争的延迟对战：**

| 阶段 | 时长 | 玩家操作 |
|------|------|----------|
| 布阵期 | 30秒 | 放置进攻/防守单位 |
| 行进期 | 60秒 | 无法操作，单位自动行进 |
| 结算期 | 10秒 | 展示战斗结果 |

**战斗单位：**
- 从卡牌中召唤
- 不同单位有不同行进速度/攻击范围
- 终点判定胜负

### 6.2 匹配机制

- 按"旅程阶段"匹配
- 相近阶段玩家对战
- 匹配等待时间：30秒 ~ 5分钟

### 6.3 奖励系统

| 结果 | 金币奖励 | 荣誉点数 |
|------|----------|----------|
| 胜利 | 500 | +20 |
| 失败 | 200 | +5 |
| 平局 | 300 | +10 |

**荣誉点数：**
- 累计荣誉可升级PVP段位
- 段位奖励：专属卡牌/头像框/特效

## 十、技术实现优先级

### Phase 1：核心玩法（MVP）

| 功能 | 优先级 | 工作量 |
|------|--------|--------|

### 11.3 战斗配置

**伤害计算公式：**
```
实际伤害 = max(1, 攻击方攻击力 - 防御方防御力 * 0.5)

暴击伤害 = 攻击力 * 2.0 (暴击率 = 智慧属性 / 100)

最终伤害 = 实际伤害 * (暴击? 2.0 : 1.0) * 技能倍率
```

**普通敌人：**
- 数量：1-5
- 强度：10-50
- HP：10 + 强度 * 2
- 伤害：3 + 强度 / 10
- 防御：5 + 强度 / 20

**精英敌人：**
- 数量：1-3
- 强度：普通 * 2
- HP：普通 * 3
- 伤害：普通 * 1.5
- 防御：普通 * 1.5
- 特殊：可能有技能

**BOSS敌人：**
- 数量：1
- 强度：100+
- HP：1000+
- 伤害：50+
- 防御：30+
- 特殊：多个阶段/技能

### 11.3.1 战斗属性计算公式

```
玩家攻击力 = 基础攻击 + 武器伤害加成 + 技能加成
玩家防御力 = 基础防御 + 防具防御 - 装备速度惩罚
玩家速度 = 基础速度 * (1 + 坐骑速度加成) * (1 - 防具速度惩罚)

伤害减免 = 防御力 / (防御力 + 100)
实际受到伤害 = 敌人伤害 * (1 - 伤害减免)
```

### 11.3.2 升级经验曲线

```
升级所需经验 = floor(10 * (当前等级 ^ 1.5))

等级1→2：10 exp
等级5→6：56 exp
等级10→11：316 exp
等级20→21：894 exp
等级50→51：3536 exp
等级100→101：10000 exp

每场战斗获得经验 = 基础经验(20) * 敌人强度/10 * 难度系数(普通1.0/精英2.0/BOSS5.0)
```

### 11.3.3 暴击与属性关系

```
暴击率 = min(50%, 智慧属性 / 100 + 装备暴击率加成)
暴击伤害 = 攻击力 * 2.0
暴击效果加成 = 装备暴击伤害加成(如有)
```
