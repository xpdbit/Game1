## Wave 1: CombatStateMachine集成

### 状态机设计（依据Oracle审查修正）
状态机应驱动战斗流程，而非被调用方轮询：
```
PlayerTurn → 执行玩家行动 → Animating(等待动画回调)
                                         ↓ (回调完成)
                                   EnemyTurn → 执行敌人行动 → Animating
                                                                  ↓ (回调完成)
                                                            PlayerTurn(循环)
```

### 文件变更
- **修改** `CombatSystem.cs` - 替换ExecuteMultiEnemyCombat直接循环为状态机驱动
- **新建** `CombatStateMachineIntegration.cs` - 桥接类，状态机驱动战斗流程
- **新建** `Assets/Tests/EditMode/CombatStateMachineIntegrationTests.cs`

### 测试用例
1. 状态机转换: Idle→Preparing→PlayerTurn→Animating→EnemyTurn→Animating循环
2. 命令队列集成: 现有CombatCommandQueue撤销/重做正常工作
3. 战斗结束: Victory/Defeat正确触发

## Wave 2: 队伍多敌战斗

### 角色体系（依据Momus审查修正）
使用现有JobType而非自创Tank/DPS/Healer：
- **Escort(镖师)**: +15%战斗加成(已有JobSystem.GetCombatBonus)
- **Healer(医者)**: +50%治疗加成(已有)
- **Merchant(商贾)**: +20%交易加成
- **Scholar(学者)**: +30%探索加成

### 文件变更
- **修改** `CombatSystem.cs` - ExecuteMultiEnemyCombat使用playerTeam

- **修改** `TeamManager.cs` - 增加战斗辅助方法
- **新建** `Assets/Tests/EditMode/CombatSystemMultiEnemyTeamTests.cs`

### 测试用例

2. 死亡队员跳过回合
