## 一、游戏概述

### 1.1 项目定位

**桌面旅者**是一款Unity 6开发的挂机放置游戏，核心差异化为：

- 透明悬浮窗：UniWindowController + D3D11，桌面融合体验
- 虚实结合：键盘/鼠标输入转化为游戏内脚程
- 分支叙事：Roguelike事件树，多结局体验
- 延迟PVP：类似皇室战争的异步对战

### 1.2 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| Unity | 6000.1.5f1 | 引擎 |
| URP | 17.1.0 | 渲染管线 |
| C# | 9.0 | 脚本语言 |
| .NET | 4.7.1 | .NET Framework |
| Input System | 1.14.0 | 新输入系统 |
| UniWindowController | 0.9.8 | 透明悬浮窗 |
| TextMeshPro | (bundled) | UI文本 |

### 1.3 游戏愿景

> "一张铺在桌面上的古代卷轴，一位行商的传奇之旅。键盘敲击化作战马蹄声，鼠标移动化作校准方向，在透明的悬浮窗中，你的商队正穿越千年风沙。"

**核心体验关键词**：

- **沉浸感**：悬浮窗如输入法般安静陪伴，不打断工作
- **成就感**：指数级成长曲线，每次上线都有惊喜
- **策略性**：资源分配、路线选择、队伍配置
- **故事性**：分支叙事事件，每趟旅程都是独特故事
- **收集欲**：卡牌收集、角色收集、成就收集

## 二、核心玩法循环

### 2.1 宏观循环架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         核心循环三层                                 │
├─────────────────────────────────────────────────────────────────────┤
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐          │
│   │   短循环    │ ──▶ │   中循环    │ ──▶ │   长循环    │          │
│   │  (分钟级)   │     │  (小时级)   │     │  (天级)     │          │
│   └─────────────┘     └─────────────┘     └─────────────┘          │
│         │                   │                   │                  │
│         ▼                   ▼                   ▼                  │
│   键盘/鼠标输入        节点旅行事件         旅程阶段/Prestige        │
│   触发即时事件        资源获取/消耗        角色成长/重置             │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 短循环：虚实交互（分钟级）

**触发条件**：玩家键盘输入或鼠标移动

| 输入类型 | 转化效率 | 额外效果 |
|----------|----------|----------|
| 键盘敲击(WASD/方向键) | 每10次 = 1秒脚程 | 连击加成（10次/秒+ = 1.5x） |
| 鼠标移动 | 每100px = 0.1秒脚程 | 精准校准（静止>2秒后移动 = 2x） |
| 滚轮滚动 | 每次 = 0.5秒脚程 | 无 |

### 2.3 中循环：节点旅行（小时级）

**节点类型与事件**：

| 节点类型 | 出现概率 | 事件类型 | 预期收益 |
|----------|----------|----------|----------|
| 起点 | 固定 | 无 | - |
| 古城(City) | 15% | NPC遭遇/商店 | 中等（金币/道具） |
| 集市(Market) | 15% | 交易事件 | 低（但稳定） |
| 荒野(Wilderness) | 20% | 战斗/随机 | 高波动 |
| 副本(Dungeon) | 25% | 连续战斗 | 高（但耗时） |
| BOSS | 10% | 极限战斗 | 极高（一次性） |
| 事件点(Event) | 10% | 分支叙事 | 极高（看选择） |
| 终点 | 固定 | 阶段结算 | - |

### 2.4 长循环：旅程与重置（天级）

**Prestige系统（轮回）**：
- 每次到达终点后，可选择"轮回"
- 轮回消耗所有资源，保留：
  - 角色等级（部分）
  - 已解锁的被动技能
  - 成就进度
  - 卡牌收藏
- 轮回奖励：轮回点数，可兑换强力永久buff

## 四、资源与经济系统

### 4.1 资源类型

| 资源 | 用途 | 获取方式 | 消耗场景 |
|------|------|----------|----------|
| 金币(Gold) | 通用货币 | 战斗/交易/事件 | 交易/招募/强化 |
| 脚程(Steps) | 旅行燃料 | 时间积累/键盘输入 | 节点移动 |
| 食物(Food) | HP恢复 | 购买/事件 | 旅行消耗 |
| 疲劳度(Energy) | 行动限制 | 时间恢复/休息 | 高难度事件 |
| 轮回点数(Prestige) | 永久加成 | 轮回重置 | 技能/buff |

### 4.2 离线收益

**计算公式**：
```
离线收益 = 在线收益 × 时间系数 × 离线加成

时间系数：
- <1小时：1.0x
- 1-6小时：0.8x
- 6-24小时：0.5x
- >24小时：0.2x

离线加成：来自被动技能和Prestige buff
```

## 六、活跃度系统

### 6.1 有效活跃度计算

```
有效活跃度 = 操作混合度 × 间隔差系数

操作混合度：
- 多种操作类型（点击、移动等）的组合
- 类型越多，混合度越高
- 公式：混合度 = 不同操作类型数 / 总操作数

间隔差系数：
- 操作时间间隔的变化程度
- 间隔变化大（忽快忽慢）= 高间隔差
```

### 6.2 活跃/非活跃优势

| 玩家类型 | 活跃度特征 | 优势 |
|----------|------------|------|
| 活跃玩家 | 高混合度 + 高间隔差 | 更多脚程、更多互动事件 |
| 非活跃玩家 | 低混合度 + 低间隔差 | 自动战斗效率+50%、离线收益+100% |

## 七、战斗系统

### 7.1 伤害计算公式

```
实际伤害 = max(1, 攻击方攻击力 - 防御方防御力 * 0.5)

暴击伤害 = 攻击力 * 2.0 (暴击率 = 智慧属性 / 100)

最终伤害 = 实际伤害 * (暴击? 2.0 : 1.0) * 技能倍率
```

### 7.2 暴击与属性关系

```
暴击率 = min(50%, 智慧属性 / 100 + 装备暴击率加成)
暴击伤害 = 攻击力 * 2.0
暴击效果加成 = 装备暴击伤害加成(如有)
```
